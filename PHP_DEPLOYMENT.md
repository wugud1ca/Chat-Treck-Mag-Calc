# Руководство по развертыванию на PHP-хостинге 🚀

Этот проект разработан на современном стеке **React + TypeScript + Tailwind CSS**. По умолчанию в среде разработки используется сервер Node.js (Vite + Express). 

Однако, если вам необходимо запустить этот готовый сайт на обычном **PHP-хостинге** (Apache / Nginx / cPanel / ISPmanager / Beget и др.), мы подготовили готовое решение, которое позволит сохранить интерактивный ИИ-чат, калькулятор и весь функционал!

---

## 📋 Как это работает?

1. **Фронтенд (React):** При сборке (`npm run build`) Vite компилирует весь ваш React-код, стили и картинки в статические файлы в папку `dist/` (это файлы `.html`, `.js`, `.css`).
2. **Бэкэнд (PHP):** Мы создали специальный прокси-скрипт `api.php`. Он принимает запросы от ИИ-чата и отправляет их напрямую в официальный **Google Gemini API** на сервере.
3. **Маршрутизация (.htaccess):** Файл `.htaccess` незаметно перенаправляет запросы с адреса `/api/chat` (который запрашивает React) на ваш новый скрипт `api.php`.

## 🛠 Пошаговая инструкция по установке

### Шаг 1. Соберите проект в статические файлы
Если вы вносите изменения в код в AI Studio, просто выполните сборку (она происходит автоматически при каждом сохранении или по кнопке):
Папка сборки появится в директории `dist/`.

### Шаг 2. Скопируйте PHP-файлы
Возьмите файлы из папки `/php-hosting/` в этом проекте и скопируйте их прямо внутрь вашей скомпилированной папки `dist/`:
- Файл `api.php` ➔ положить в `dist/api.php`
- Файл `.htaccess` ➔ положить в `dist/.htaccess`

*После этого папка `dist` будет содержать:*
```text
dist/
├── assets/         (JS- и CSS-файлы сайта)
├── index.html      (главная страница)
├── api.php         (наш обработчик ИИ-чата)
└── .htaccess       (правила перенаправления запросов)
```

### Шаг 3. Загрузите файлы на хостинг
Загрузите всё содержимое папки `dist` в корневую директорию вашего сайта на хостинге (обычно это папка `public_html`, `www` или `httpdocs`) через FTP-клиент (например, FileZilla) или файловый менеджер панели управления хостинга.

### Шаг 4. Установите ваш API-ключ Gemini
Чтобы ИИ-чат работал, скрипт `api.php` должен знать ваш API-ключ Gemini. Вы можете настроить его тремя способами (выберите один наиболее удобный):

#### Способ А (Самый простой и быстрый):
Откройте загруженный файл `api.php` на хостинге в текстовом редакторе, найдите строку:
```php
$gemini_api_key = "YOUR_GEMINI_API_KEY_HERE";
```
Замените `"YOUR_GEMINI_API_KEY_HERE"` на ваш настоящий ключ API Gemini (начинается на `AIzaSy...`). Сохраните файл.

#### Способ Б (Безопасный через `.env` файл):
Создайте файл `.env` в корневом каталоге вашего хостинга (рядом с `api.php`) и напишите в него:
```env
GEMINI_API_KEY=ваш_ключ_здесь
```
Наш скрипт `api.php` автоматически прочитает ключ из этого файла, а файл `.htaccess` заблокирует к нему доступ для обычных посетителей, чтобы никто не смог его скачать.

#### Способ В (Через настройки сервера):
В панели управления хостинга (например, Beget, cPanel) в разделе «Переменные окружения» (Environment Variables) добавьте переменную:
- Имя: `GEMINI_API_KEY`
- Значение: `ваш_ключ_здесь`

## 🔍 Проверка работоспособности

1. Откройте ваш домен в браузере (например, `https://my-express-site.ru`).
2. Попробуйте написать любое сообщение в ИИ-чат.
3. Если чат отвечает мгновенно и экспертно, поздравляем — всё работает идеально на вашем PHP-сервере! 🎉

### ❓ Что делать при ошибках?
* **Чат выдает ошибку сети:** Убедитесь, что на хостинге включен модуль Apache `mod_rewrite` (на 99% хостингов он включен по умолчанию) и загружен файл `.htaccess`.
* **Чат пишет, что ключ отсутствует:** Перепроверьте правильность ввода ключа в `api.php` или `.env` файле.
* **Ошибка 500 (Internal Server Error):** Убедитесь, что на вашем сервере установлена версия PHP не ниже 7.4 и включено расширение `cURL` (оно также по умолчанию включено везде для отправки запросов).
